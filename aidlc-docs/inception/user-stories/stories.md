## 고객 스토리 (Customer Stories)

### Epic 1: 고객 인증 및 세션

#### US-C01: 태블릿 자동 로그인
**As a** 식당 고객 (김민준/이영숙)
**I want to** 태블릿을 켜면 자동으로 로그인되어 있기를
**So that** 별도 로그인 절차 없이 바로 주문할 수 있다

**Acceptance Criteria:**
- [ ] 태블릿 브라우저 접속 시 저장된 인증 정보로 자동 로그인된다
- [ ] 자동 로그인 성공 시 메뉴 화면이 바로 표시된다
- [ ] 자동 로그인 실패 시 초기 설정 화면이 표시된다
- [ ] 로그인 정보는 브라우저 로컬 스토리지에 저장된다

---

### Epic 2: 메뉴 조회

#### US-C02: 카테고리별 메뉴 목록 조회
**As a** 식당 고객
**I want to** 카테고리별로 분류된 메뉴 목록을 볼 수 있기를
**So that** 원하는 종류의 메뉴를 쉽게 찾을 수 있다

**Acceptance Criteria:**
- [ ] 메뉴 화면에 카테고리 탭/버튼이 표시된다
- [ ] 카테고리 선택 시 해당 카테고리의 메뉴만 표시된다
- [ ] 각 메뉴는 카드 형태로 표시된다
- [ ] 카테고리 간 전환이 즉시 반영된다

#### US-C03: 메뉴 상세 정보 확인
**As a** 식당 고객
**I want to** 메뉴의 상세 정보(이미지, 설명, 가격)를 확인할 수 있기를
**So that** 주문 전에 메뉴에 대해 충분히 알 수 있다

**Acceptance Criteria:**
- [ ] 메뉴 카드에 메뉴명, 가격, 이미지가 표시된다
- [ ] 메뉴 카드 탭 시 상세 정보(설명 포함)가 표시된다
- [ ] 이미지가 없는 메뉴는 기본 플레이스홀더 이미지가 표시된다
- [ ] 가격은 원화(₩) 형식으로 표시된다

#### US-C04: 카테고리 빠른 이동
**As a** 식당 고객
**I want to** 카테고리 간 빠르게 이동할 수 있기를
**So that** 여러 카테고리의 메뉴를 효율적으로 탐색할 수 있다

**Acceptance Criteria:**
- [ ] 카테고리 네비게이션이 화면 상단에 고정 표시된다
- [ ] 현재 선택된 카테고리가 시각적으로 강조된다
- [ ] 카테고리 탭 시 해당 메뉴 목록으로 즉시 전환된다
- [ ] 터치 타겟이 44x44px 이상이다

### Epic 3: 장바구니 관리

#### US-C05: 메뉴를 장바구니에 추가
**As a** 식당 고객
**I want to** 원하는 메뉴를 장바구니에 추가할 수 있기를
**So that** 여러 메뉴를 모아서 한번에 주문할 수 있다

**Acceptance Criteria:**
- [ ] 메뉴 카드 또는 상세 화면에서 "추가" 버튼을 탭하면 장바구니에 추가된다
- [ ] 추가 시 시각적 피드백이 표시된다 (토스트/애니메이션)
- [ ] 이미 장바구니에 있는 메뉴 추가 시 수량이 증가한다
- [ ] 장바구니 아이콘에 현재 항목 수가 표시된다

#### US-C06: 장바구니 수량 조절
**As a** 식당 고객
**I want to** 장바구니에 담긴 메뉴의 수량을 조절할 수 있기를
**So that** 원하는 만큼 정확하게 주문할 수 있다

**Acceptance Criteria:**
- [ ] +/- 버튼으로 수량을 증가/감소할 수 있다
- [ ] 수량이 1 미만으로 감소하면 해당 메뉴가 장바구니에서 삭제된다
- [ ] 수량 변경 시 총 금액이 즉시 재계산된다
- [ ] 버튼 크기가 터치 친화적이다 (44x44px 이상)

#### US-C07: 장바구니 항목 삭제
**As a** 식당 고객
**I want to** 장바구니에서 특정 메뉴를 삭제할 수 있기를
**So that** 실수로 추가한 메뉴를 제거할 수 있다

**Acceptance Criteria:**
- [ ] 각 장바구니 항목에 삭제 버튼이 있다
- [ ] 삭제 시 해당 항목이 즉시 제거된다
- [ ] 삭제 후 총 금액이 재계산된다
- [ ] 모든 항목 삭제 시 빈 장바구니 안내가 표시된다

#### US-C08: 장바구니 비우기
**As a** 식당 고객
**I want to** 장바구니를 한번에 비울 수 있기를
**So that** 처음부터 다시 메뉴를 선택할 수 있다

**Acceptance Criteria:**
- [ ] "장바구니 비우기" 버튼이 있다
- [ ] 비우기 전 확인 메시지가 표시된다
- [ ] 확인 시 모든 항목이 삭제된다
- [ ] 비운 후 빈 장바구니 안내가 표시된다

#### US-C09: 장바구니 총 금액 확인
**As a** 식당 고객
**I want to** 장바구니의 총 주문 금액을 실시간으로 확인할 수 있기를
**So that** 예산 내에서 주문할 수 있다

**Acceptance Criteria:**
- [ ] 장바구니 하단에 총 금액이 항상 표시된다
- [ ] 항목 추가/삭제/수량 변경 시 즉시 업데이트된다
- [ ] 금액은 원화(₩) 형식으로 표시된다
- [ ] 각 항목별 소계(단가 x 수량)도 표시된다

#### US-C10: 장바구니 데이터 유지
**As a** 식당 고객
**I want to** 페이지를 새로고침해도 장바구니 내용이 유지되기를
**So that** 실수로 새로고침해도 다시 담을 필요가 없다

**Acceptance Criteria:**
- [ ] 장바구니 데이터가 로컬 스토리지에 저장된다
- [ ] 페이지 새로고침 시 장바구니가 복원된다
- [ ] 브라우저 탭을 닫았다 열어도 유지된다
- [ ] 주문 완료 시에만 장바구니가 비워진다

### Epic 4: 주문 생성

#### US-C11: 주문 내역 최종 확인
**As a** 식당 고객
**I want to** 주문 확정 전에 주문 내역을 최종 확인할 수 있기를
**So that** 실수 없이 정확하게 주문할 수 있다

**Acceptance Criteria:**
- [ ] 주문 확인 화면에 모든 메뉴, 수량, 단가가 표시된다
- [ ] 총 주문 금액이 명확하게 표시된다
- [ ] "주문 확정" 버튼과 "돌아가기" 버튼이 있다
- [ ] 테이블 번호가 표시된다

#### US-C12: 주문 확정
**As a** 식당 고객
**I want to** 주문을 확정하여 매장에 전달할 수 있기를
**So that** 음식을 받을 수 있다

**Acceptance Criteria:**
- [ ] "주문 확정" 버튼 탭 시 서버에 주문이 전송된다
- [ ] 주문 전송 중 로딩 표시가 나타난다
- [ ] 주문 성공 시 주문 번호가 표시된다
- [ ] 주문 성공 후 5초 뒤 메뉴 화면으로 자동 리다이렉트된다
- [ ] 주문 성공 시 장바구니가 자동으로 비워진다

#### US-C13: 주문 실패 처리
**As a** 식당 고객
**I want to** 주문 실패 시 명확한 안내를 받을 수 있기를
**So that** 다시 시도하거나 도움을 요청할 수 있다

**Acceptance Criteria:**
- [ ] 주문 실패 시 에러 메시지가 표시된다
- [ ] 장바구니 내용이 유지된다
- [ ] 다시 시도할 수 있는 버튼이 있다
- [ ] 네트워크 오류와 서버 오류를 구분하여 안내한다

---

### Epic 5: 주문 내역 조회

#### US-C14: 현재 세션 주문 목록 조회
**As a** 식당 고객
**I want to** 현재 식사 중 주문한 내역을 확인할 수 있기를
**So that** 무엇을 주문했는지, 총 얼마인지 파악할 수 있다

**Acceptance Criteria:**
- [ ] 주문 내역 탭/버튼으로 주문 목록에 접근할 수 있다
- [ ] 현재 테이블 세션의 주문만 표시된다
- [ ] 이전 고객의 주문은 표시되지 않는다
- [ ] 주문 시간 순으로 정렬된다

#### US-C15: 주문 상세 정보 확인
**As a** 식당 고객
**I want to** 각 주문의 상세 정보를 확인할 수 있기를
**So that** 주문 상태와 내용을 정확히 알 수 있다

**Acceptance Criteria:**
- [ ] 각 주문에 주문 번호가 표시된다
- [ ] 주문 시각이 표시된다
- [ ] 주문 메뉴 및 수량이 표시된다
- [ ] 주문 금액이 표시된다
- [ ] 주문 상태(대기중/준비중/완료)가 표시된다

#### US-C16: 주문 상태 시각적 구분
**As a** 식당 고객
**I want to** 주문 상태를 색상/아이콘으로 쉽게 구분할 수 있기를
**So that** 한눈에 어떤 주문이 준비 중인지 알 수 있다

**Acceptance Criteria:**
- [ ] 대기중/준비중/완료 상태가 서로 다른 색상으로 표시된다
- [ ] 상태 텍스트와 함께 아이콘이 표시된다
- [ ] 색상 대비가 충분하여 구분이 명확하다

## 관리자 스토리 (Admin Stories)

### Epic 6: 관리자 인증

#### US-A01: 관리자 로그인
**As a** 매장 관리자 (박성호/최수진)
**I want to** 매장 식별자와 계정 정보로 로그인할 수 있기를
**So that** 매장 관리 시스템에 접근할 수 있다

**Acceptance Criteria:**
- [ ] 매장 식별자, 사용자명, 비밀번호 입력 필드가 있다
- [ ] 로그인 성공 시 대시보드 화면으로 이동한다
- [ ] 로그인 실패 시 에러 메시지가 표시된다
- [ ] 비밀번호는 마스킹 처리된다

#### US-A02: 세션 유지
**As a** 매장 관리자
**I want to** 16시간 동안 세션이 유지되기를
**So that** 영업 시간 동안 반복 로그인할 필요가 없다

**Acceptance Criteria:**
- [ ] 로그인 후 16시간 동안 세션이 유지된다
- [ ] 브라우저 새로고침 시에도 세션이 유지된다
- [ ] 16시간 후 자동으로 로그아웃된다
- [ ] 자동 로그아웃 시 로그인 화면으로 리다이렉트된다

#### US-A03: 로그인 시도 제한
**As a** 시스템
**I want to** 로그인 시도 횟수를 제한하기를
**So that** 무차별 대입 공격을 방지할 수 있다

**Acceptance Criteria:**
- [ ] 연속 5회 로그인 실패 시 계정이 일시 잠금된다
- [ ] 잠금 시 남은 대기 시간이 표시된다
- [ ] 일정 시간 후 자동으로 잠금이 해제된다

### Epic 7: 실시간 주문 모니터링

#### US-A04: 테이블별 주문 대시보드 조회
**As a** 매장 관리자 (박성호/최수진)
**I want to** 테이블별 주문 현황을 그리드 형태로 볼 수 있기를
**So that** 매장 전체 주문 상황을 한눈에 파악할 수 있다

**Acceptance Criteria:**
- [ ] 테이블별 카드가 그리드 레이아웃으로 표시된다
- [ ] 각 카드에 테이블 번호가 표시된다
- [ ] 각 카드에 총 주문액이 표시된다
- [ ] 각 카드에 최신 주문 미리보기가 표시된다
- [ ] 주문이 없는 테이블은 빈 상태로 표시된다

#### US-A05: 실시간 신규 주문 수신
**As a** 매장 직원 (최수진)
**I want to** 새로운 주문이 들어오면 즉시 화면에 표시되기를
**So that** 주문을 놓치지 않고 빠르게 대응할 수 있다

**Acceptance Criteria:**
- [ ] 신규 주문이 2초 이내에 화면에 표시된다
- [ ] 신규 주문 카드가 시각적으로 강조된다 (색상 변경/애니메이션)
- [ ] SSE 연결이 유지되어 실시간 업데이트가 된다
- [ ] SSE 연결 끊김 시 자동 재연결된다

#### US-A06: 주문 상세 보기
**As a** 매장 관리자
**I want to** 테이블 카드를 클릭하여 전체 주문 목록을 볼 수 있기를
**So that** 해당 테이블의 상세 주문 내역을 확인할 수 있다

**Acceptance Criteria:**
- [ ] 테이블 카드 클릭 시 상세 주문 목록이 표시된다
- [ ] 각 주문의 메뉴명, 수량, 금액이 표시된다
- [ ] 주문 시각과 주문 번호가 표시된다
- [ ] 닫기 버튼으로 대시보드로 복귀할 수 있다

#### US-A07: 주문 상태 변경
**As a** 매장 직원 (최수진)
**I want to** 주문 상태를 변경할 수 있기를 (대기중 → 준비중 → 완료)
**So that** 주문 처리 진행 상황을 추적할 수 있다

**Acceptance Criteria:**
- [ ] 각 주문에 상태 변경 버튼이 있다
- [ ] 대기중 → 준비중 → 완료 순서로 변경 가능하다
- [ ] 상태 변경 시 즉시 화면에 반영된다
- [ ] 상태 변경 시 고객 화면에도 반영된다

#### US-A08: 테이블별 필터링
**As a** 매장 관리자
**I want to** 특정 테이블의 주문만 필터링하여 볼 수 있기를
**So that** 특정 테이블에 집중하여 관리할 수 있다

**Acceptance Criteria:**
- [ ] 테이블 번호로 필터링할 수 있는 기능이 있다
- [ ] 필터 적용 시 해당 테이블 카드만 표시된다
- [ ] 필터 해제 시 전체 테이블이 다시 표시된다

### Epic 8: 테이블 관리

#### US-A09: 테이블 초기 설정
**As a** 매장 사장님 (박성호)
**I want to** 테이블 번호와 비밀번호를 설정할 수 있기를
**So that** 태블릿이 해당 테이블로 자동 로그인할 수 있다

**Acceptance Criteria:**
- [ ] 테이블 번호 입력 필드가 있다
- [ ] 테이블 비밀번호 설정 필드가 있다
- [ ] 설정 완료 시 16시간 세션이 생성된다
- [ ] 설정 성공/실패 피드백이 표시된다
