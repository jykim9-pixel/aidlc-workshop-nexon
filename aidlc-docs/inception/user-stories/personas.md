# 테이블오더 서비스 - 페르소나 정의

## 고객 페르소나 (Customer Personas)

### Persona 1: 김민준 (디지털 네이티브 고객)

| 항목 | 내용 |
|------|------|
| **이름** | 김민준 |
| **나이** | 28세 |
| **직업** | IT 회사 직원 |
| **디지털 친숙도** | 높음 |
| **특징** | 스마트폰/태블릿 사용에 익숙, 빠른 조작 선호 |

**행동 패턴:**
- 메뉴를 빠르게 스크롤하며 탐색
- 카테고리 이동을 자주 사용
- 장바구니에 여러 메뉴를 한번에 추가
- 주문 내역을 통해 추가 주문 여부 판단

**니즈:**
- 빠른 응답 속도
- 직관적인 네비게이션
- 최소한의 탭으로 주문 완료

**불만 요소:**
- 느린 로딩
- 불필요한 확인 단계
- 복잡한 UI 구조

### Persona 2: 이영숙 (디지털 비친숙 고객)

| 항목 | 내용 |
|------|------|
| **이름** | 이영숙 |
| **나이** | 58세 |
| **직업** | 자영업 |
| **디지털 친숙도** | 낮음 |
| **특징** | 태블릿 사용 경험 적음, 큰 글씨/버튼 선호 |

**행동 패턴:**
- 메뉴를 천천히 하나씩 확인
- 이미지를 보고 메뉴 선택
- 수량 조절 시 실수 가능성 있음
- 주문 확정 전 여러 번 확인

**니즈:**
- 큰 터치 타겟 (44x44px 이상)
- 명확한 시각적 피드백
- 실수 방지 (확인 단계)
- 간단하고 명확한 흐름

**불만 요소:**
- 작은 버튼
- 복잡한 제스처 (스와이프 등)
- 피드백 없는 동작
- 되돌리기 어려운 조작

## 관리자 페르소나 (Admin Personas)

### Persona 3: 박성호 (매장 사장님)

| 항목 | 내용 |
|------|------|
| **이름** | 박성호 |
| **나이** | 45세 |
| **직업** | 식당 사장 |
| **디지털 친숙도** | 중간 |
| **특징** | 매장 전체 운영 책임, 메뉴/테이블 설정 담당 |

**행동 패턴:**
- 영업 시작 전 메뉴 설정 및 테이블 초기화
- 영업 중 간헐적으로 주문 현황 확인
- 영업 종료 시 테이블 이용 완료 처리
- 메뉴 가격/구성 변경 시 메뉴 관리 사용

**니즈:**
- 매장 전체 현황 한눈에 파악
- 메뉴 등록/수정이 간편해야 함
- 테이블 설정이 직관적이어야 함
- 과거 주문 내역 조회 (매출 파악용)

**불만 요소:**
- 복잡한 설정 과정
- 전체 현황 파악이 어려운 UI
- 느린 반응 속도

### Persona 4: 최수진 (매장 직원)

| 항목 | 내용 |
|------|------|
| **이름** | 최수진 |
| **나이** | 22세 |
| **직업** | 식당 아르바이트 |
| **디지털 친숙도** | 높음 |
| **특징** | 주문 처리 및 서빙 담당, 빠른 대응 필요 |

**행동 패턴:**
- 실시간 주문 모니터링 상시 확인
- 신규 주문 확인 후 주방에 전달 (구두)
- 주문 상태를 "준비중" → "완료"로 변경
- 잘못된 주문 삭제 처리

**니즈:**
- 신규 주문 즉시 인지 (시각적 강조)
- 빠른 상태 변경 (최소 탭)
- 테이블별 주문 현황 빠른 파악
- 실수 시 되돌리기 가능

**불만 요소:**
- 신규 주문 놓침
- 상태 변경에 많은 단계 필요
- 어떤 테이블 주문인지 구분 어려움
